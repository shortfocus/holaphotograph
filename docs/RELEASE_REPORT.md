## 1. 작업 개요

올라포토 카메라 리뷰 웹사이트의 디자인 개선 및 기능 강화를 진행했습니다.  
디자인 스펙(DESIGN_IMPROVEMENT.md)을 기반으로 UI/UX 전반을 업데이트하고, 콘텐츠 노출 로직을 섹션별로 최적화하였습니다.

---

## 2. 디자인 개선 사항

### 2.1 히어로 섹션

- **캐러셀 구조**: 단일 슬라이드 → 3슬라이드 캐러셀 (메인 소개 / 프로모션 / 고객 리뷰 안내)
- **레이아웃**: 좌측 텍스트 + 우측 영상
- **카피**: "카메라 선택부터 촬영 노하우까지, 실전 기반 리뷰 플랫폼"
- **CTA**: 채널 아이콘(호버 시 원본 색상), 버튼 제거로 심플화
- **높이**: 뷰포트 전체(min-h-screen) 적용

### 2.2 전문가 소개 섹션

- **프로필 이미지**: 로고(logo.png) 원형 프로필 적용
- **실적 카드**: 유튜브 구독자, 네이버 블로그 방문자 수 표시
- **주요 콘텐츠 태그**: 장비 언박싱 & 리뷰, 실사용 비교 분석, 구매 가이드, 촬영 노하우
- **SNS 링크**: 유튜브, 네이버 블로그, 인스타그램 아이콘

### 2.3 섹션 타이틀 및 설명 통일

| 섹션                  | 타이틀                  | 설명 문장                               |
| --------------------- | ----------------------- | --------------------------------------- |
| 실전 영상 리뷰        | 📺 실전 영상 리뷰       | 직접 사용하고 비교한 장비 리뷰 영상     |
| 빠르게 보는 장비 팁   | 🎬 빠르게 보는 장비 팁  | 1분 안에 핵심만 정리한 장비 팁          |
| 장비 비교 & 상세 분석 | ✍ 장비 비교 & 상세 분석 | 구매 전 꼭 확인해야 할 비교 분석 콘텐츠 |

### 2.4 섹션 배경 패턴

- **흰색(#FFFFFF)** / **연한 회색(#F8FAFC)** 2색 교차로 시각적 계층 강화
- 강의 티저 섹션: 네이비(#0E172A) 배경, 협업 섹션과의 경계선 제거

### 2.5 성과 지표 섹션

- "누적 성과" 라벨 추가
- 숫자 카운트업 애니메이션 적용
- 유튜브 구독자, 월간 조회수, 누적 방문자 표시

### 2.6 푸터

- **카피**: "장비 선택부터 촬영까지, 실전 기반 가이드."
- **배경**: #0F172A (히어로와 동일)
- 채널 아이콘: 호버 시 원본 색상, 크기 확대

### 2.7 폰트

- Pretendard 웹폰트 적용 (CDN)

## 3. 기능 개선 사항

### 3.1 실전 영상 리뷰 (유튜브 롱폼)

- **키워드 매칭**: 제목에 "리뷰", "언박싱", "비교", "사용기", "후기" 등 포함 시 상단 우선 정렬
- 섹션 타이틀에 맞는 영상이 먼저 노출되도록 개선

### 3.2 빠르게 보는 장비 팁 (Shorts)

- **키워드 매칭**: 제목에 "팁", "꿀팁" 등 포함 시 상단 우선 정렬
- **60초 이하 Shorts**만 노출
- **[대여] 영상 제외**: 제목에 "[대여]" 포함 영상은 노출하지 않음

### 3.3 장비 비교 & 상세 분석 (네이버 블로그)

- **콘텐츠 필터**: 제목·카테고리에 "렌즈 리뷰" 또는 "카메라 리뷰" 포함된 글만 노출
- 섹션 타이틀에 맞는 블로그 콘텐츠만 표시

### 3.4 강의 소식 받기

- **기존**: mailto 링크 (사용자가 직접 이메일 발송)
- **변경**: 이메일 입력 폼 → DB(D1) 저장
- 사용자가 이메일만 입력하면 강의 소식 수신 신청 완료
- 중복 등록 시 "이미 등록된 이메일입니다." 안내
- 성공/실패 메시지 표시

## 4. 기술 변경 사항

### 4.1 API (Worker)

- `POST /api/lecture-signup`: 강의 소식 수신 이메일 저장
- 네이버 RSS 응답에 `compare` 배열 추가 (장비 비교 & 상세 분석용)
- 유튜브 검색/정렬 로직 개선 (키워드 매칭, [대여] 제외)

### 4.2 데이터베이스

- `lecture_signups` 테이블 추가 (이메일, 등록일시)
- 마이그레이션: `worker/migrations/0003_lecture_signups.sql`

### 4.3 프론트엔드

- 강의 섹션: 이메일 입력 폼 + 제출 버튼
- API 클라이언트: `submitLectureSignup()` 함수 추가

## 5. 수집 데이터 관리

### 강의 소식 수신 이메일

- **저장 위치**: Cloudflare D1 `lecture_signups` 테이블
- **조회 방법**: Cloudflare 대시보드 → D1 → camera-review-db → lecture_signups 테이블
- 또는 wrangler CLI로 조회 가능
